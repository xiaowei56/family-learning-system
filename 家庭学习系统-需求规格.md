# 家庭学习系统 — 需求规格说明书

> 版本：v1.0
> 更新日期：2026-06-21

---

## 1. 项目概述

面向**高中、初中、小学**三个阶段学生的家庭学习辅助系统，部署在家庭 NAS（局域网内使用，不连接公网），客户端支持 **Web** 和 **Android App**。

**部署环境：** 纯内网使用，不加密，不暴露公网。

---

## 2. 核心功能模块

### 2.1 学习记录与记忆曲线提醒

| 项目 | 说明 |
|------|------|
| 功能 | 记录每天的学习内容和学习笔记 |
| 提醒机制 | 基于**艾宾浩斯遗忘曲线**，自动计算并提醒当天需要复习的内容 |
| 输入 | 学习日期、科目、知识点标题、学习笔记（富文本） |
| 输出 | 当天待复习列表推送 |

**实现要点：**
- 学习记录按**科目**分类管理，每个科目有独立的复习计划
- 艾宾浩斯遗忘曲线复习时间点：1天、2天、4天、7天、15天、30天、90天…
- 用户可**手动标记"已掌握"**，提前结束该知识点的复习周期
- 用户可调整复习频率（快 / 中 / 慢三档）
- 当日复习列表按科目分组展示，便于集中复习同一科目

### 2.2 试卷与错题拍照识别

| 项目 | 说明 |
|------|------|
| 功能 | 上传照片记录试卷和错题，AI 自动判断题目是否正确 |
| 用户交互 | 可查询解题过程，**优先用图示来理解题目** |
| 输入 | 拍照图片（试卷 / 错题本） |
| 输出 | 题目内容、正误判断结果、解题步骤、图示解析 |

**图示生成规则：**

| 题型/科目 | 图示方式 |
|-----------|---------|
| 数学几何题 | **SVG 图形**（三角形、圆、坐标系、辅助线标注） |
| 函数/代数题 | **公式渲染（MathJax）+ 图表（Chart.js 曲线图）** |
| 物理/化学题 | **流程图或示意图**（受力分析图、电路图、化学实验装置图） |

**实现要点：**
- OCR 使用 **PaddleOCR**（本地部署，CPU 可运行）
- AI 推理使用本地 API：`http://192.168.110.7/v1`（兼容 OpenAI API 格式）
- 错题自动归集到错题本
- 支持多科目（数学、物理、化学、语文、英语等）

### 2.3 智能举一反三

| 项目 | 说明 |
|------|------|
| 功能 | 根据用户的错题，自动生成同类题目供练习 |
| 附加 | 给出此类题目的**通用解题思路** |
| 输入 | 错题内容 + 科目 + 知识点 |
| 输出 | 类似题 3～5 道 + 解题方法总结（步骤化 + 图示化） |

**实现要点：**
- 同知识点、同题型、难度递进
- 解题思路结构化：分步骤说明 + 结合图示

### 2.4 考试成绩追踪

| 项目 | 说明 |
|------|------|
| 功能 | 记录考试成绩，自动计算得分率（得分 / 总分） |
| 考试类型 | 日常练习、周测、月考测试、期中考试、期末考试 |
| 输入 | 科目、考试类型、得分、总分、考试日期 |
| 输出 | 按科目和类别生成**得分率曲线图** |

**实现要点：**
- 得分率 = 得分 / 总分（百分比）
- 考试成绩按**科目**独立记录，每个科目生成独立的得分率曲线图
- 图表维度：横轴 = 时间，纵轴 = 得分率
- 可按**科目**切换查看，也可按**考试类型**（日常/周测/月考/期中/期末）筛选
- 支持**多科目对比**：在同一图表中叠加显示多个科目的得分率趋势

### 2.5 试卷整理功能

| 项目 | 说明 |
|------|------|
| 功能 | 用户上传考试试卷（拍照/扫描），建立电子试卷库 |
| 核心用途 | ① 后续复习查阅 ② 擦除笔迹后重新打印练习 |
| 输入 | 试卷拍照图片 |
| 输出 | 原始试卷存档 / 去笔迹后的"空白卷"下载打印 |

**实现要点：**

**2.5.1 试卷归档整理**
- 上传试卷照片，按**科目 + 考试类型 + 日期**自动分类归档
- 支持查看、搜索、筛选历史试卷
- 试卷详情展示原始图片（可缩放、翻页）

**2.5.2 去笔迹重印**
- 利用 AI（结合 PaddleOCR + 图像修复）识别并擦除试卷上的手写笔迹
- 输出干净的"空白卷"图片，支持下载和打印
- 去笔迹质量：保留原试卷的印刷内容（题目、图表、横线），仅去除手写部分

**2.5.3 在线复习**
- 在 Web/App 上直接查看试卷原图
- 支持在试卷图片上做标记（高亮、批注），下次复习可见

**与 2.2 的关系：** 试卷整理（2.5）侧重于试卷的**存档和复用**；错题拍照（2.2）侧重于**题目识别和解题**。两者共享 OCR 能力，但功能和流程独立。从试卷整理中也可以一键跳转到错题分析。

### 2.6 智能出卷与薄弱点分析

| 项目 | 说明 |
|------|------|
| 功能 | 根据用户错题和知识薄弱点，自动生成练习试卷 |
| 附加 | 给出薄弱点的学习建议、学习方法、记忆方法 |
| 输入 | 用户错题数据、历史考试成绩、知识点掌握情况 |
| 输出 | 定制练习试卷 + 学习诊断报告 |

**2.6.1 薄弱点分析**
- 综合错题本（2.2）和考试成绩（2.4）的数据，自动识别用户的**知识薄弱点**
- 按学科、知识点维度统计错误率和得分率，量化薄弱程度
- 可视化展示：薄弱知识点雷达图/热力图，一目了然

**2.6.2 智能组卷**
- 根据薄弱点列表，从题库中（或 AI 生成）自动组卷
- 组卷策略：薄弱知识点覆盖 + 难度递进（基础→巩固→拔高）
- 每道题标注对应的知识点和考察能力
- 支持手动调整：增删题目、调整顺序

**2.6.3 学习诊断报告**
- 每份练习试卷附带诊断报告，包含：
  - 各知识点的掌握等级（薄弱 / 待巩固 / 已掌握）
  - 针对每个薄弱点的**具体学习建议**（可执行的学习步骤）
  - **学习方法推荐**（针对科目特点：如数学重练习、物理重理解概念、语文重积累）
  - **记忆方法推荐**（如：艾宾浩斯复习计划、联想记忆法、思维导图法、错题卡片等）

**与 2.3 的关系：** 举一反三（2.3）针对单道错题生成同类题目；智能出卷（2.6）从全局分析用户知识结构，综合生成整份试卷和系统学习建议。

## 4. 数据模型（核心实体）

```
User (学生/家长)
  ├── id, name, grade_level (小学/初中/高中), avatar
  ├── subjects (关联科目列表)

StudyRecord (学习记录)
  ├── id, user_id, subject, date, title, content(富文本)
  ├── tags (知识点标签，用于记忆曲线关联)

ReviewSchedule (复习计划)
  ├── id, study_record_id, review_date, review_count
  ├── status (pending / done / mastered)
  ├── speed_mode (快 fast / 中 medium / 慢 slow)

ExamResult (考试成绩)
  ├── id, user_id, subject
  ├── exam_type (日常练习 / 周测 / 月考 / 期中 / 期末)
  ├── score, total_score, date, notes

WrongProblem (错题)
  ├── id, user_id, subject, knowledge_point
  ├── image_path (原题图片), content (OCR 文本)
  ├── correct_answer, user_answer, is_correct
  ├── ai_solution (解题过程 JSON)
  ├── ai_diagram_type (svg / chart / flowchart)
  ├── ai_diagram_content (图示内容)

SimilarProblem (举一反三)
  ├── id, wrong_problem_id (关联错题)
  ├── content, answer, solution_steps, diagram
  ├── difficulty (1~5)

ExamPaper (试卷)
  ├── id, user_id, subject, exam_type
  ├── title (试卷名称), exam_date, duration (考试时长)
  ├── original_image_path (原图路径)
  ├── cleaned_image_path (去笔迹后图片路径)
  ├── status (uploaded / cleaned / archived)

PaperAnnotation (试卷批注)
  ├── id, exam_paper_id
  ├── page_number, position_x, position_y
  ├── content (批注文本), color

WeakPoint (薄弱点分析)
  ├── id, user_id, subject
  ├── knowledge_point (知识点名称)
  ├── error_rate (错误率), score_rate (得分率)
  ├── mastery_level (薄弱 / 待巩固 / 已掌握)
  ├── last_analyzed_date

PracticePaper (练习试卷)
  ├── id, user_id, subject
  ├── title, created_date, difficulty
  ├── status (draft / published / completed)
  ├── questions (JSON: 题目列表，每题含知识点、难度)

LearningAdvice (学习建议)
  ├── id, weak_point_id
  ├── suggestion_type (学习方法 / 记忆方法 / 学习步骤)
  ├── content (建议内容)
```

## 5. 分阶段交付计划

### Phase 1 — 基础设施 + 基础功能（无 AI 依赖）

**目标：** 可用的学习记录与成绩追踪系统

- [ ] Docker Compose 部署骨架（Nginx + FastAPI + PostgreSQL + MinIO）
- [ ] 用户管理（Web 注册/登录）
- [ ] 学习记录 CRUD（富文本编辑器）
- [ ] 考试成绩录入 + 得分率自动计算
- [ ] 得分率曲线图（按科目 / 考试类型筛选）
- [ ] 试卷整理基础功能：上传、归档、查看、搜索
- [ ] 响应式 Web 前端覆盖以上功能

### Phase 2 — AI 能力接入

**目标：** 错题拍照、AI 解题、图示生成

- [ ] PaddleOCR 本地部署 + API 封装
- [ ] 试卷/错题拍照上传 → OCR 识别 → 结果展示
- [ ] LLM 接入本地 `192.168.110.7/v1`，实现：
  - 题目正误判断
  - 解题步骤生成
  - 图示生成（SVG / Chart.js / MathJax / 流程图）
- [ ] 错题本展示 + 解题过程查看
- [ ] 试卷去笔迹重印（OCR 定位 + 图像修复擦除手写）
- [ ] 试卷在线查看 + 批注标记
- [ ] Android App 基础版本（Flutter）

### Phase 3 — 智能学习闭环

**目标：** 举一反三、记忆曲线提醒

- [ ] 艾宾浩斯遗忘曲线算法 + 复习提醒
- [ ] 手动标记"已掌握"功能
- [ ] 复习频率调节（快 / 中 / 慢）
- [ ] 基于错题的举一反三（同知识点 AI 出题）
- [ ] 解题思路总结（步骤化 + 图示化）
- [ ] 薄弱点分析引擎（综合错题 + 成绩数据识别薄弱点）
- [ ] 智能组卷（AI 自动生成练习试卷）
- [ ] 学习诊断报告 + 学习建议 / 学习方法 / 记忆方法推荐
- [ ] Android App 完整功能覆盖

## 7. 非功能性需求

- **部署：** Docker Compose 一键部署到 NAS
- **网络：** 纯内网，不依赖公网
- **存储：** 图片存储在 MinIO，不加密
- **AI 延迟：** 本地 LLM 推理延迟需在可接受范围（10s 内出解题结果）
- **浏览器兼容：** 现代浏览器（Chrome / Edge / Safari）
